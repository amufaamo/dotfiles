# 汎用 Agent Team Configuration
# 最終更新: 自動管理（/evolve コマンドで更新）

---

## Step 1: タスク分析（Leadが最初に必ず実行）

タスクを受け取ったら、チームを組む前に以下を自問すること：

### 1-A. 作業種別は何か？
- **実装系**: コード・インフラ・設定を作る
- **調査系**: 情報収集・文献調査・技術検証
- **執筆系**: 論文・レポート・ドキュメント・記事
- **分析系**: データ解析・ログ調査・パフォーマンス計測
- **レビュー系**: コード・文章・設計の評価

### 1-B. 並列化できるか？
- 独立した「関心事」が3つ以上ある → Team有効
- 前の作業が終わらないと次に進めない → Team不要
- 同じファイル・文書を複数人が同時編集 → Team不要

### 1-C. 規模の判定
- 数時間で終わる小タスク → 単独
- 半日〜1日規模 → 2〜3名
- 複数日・複数領域 → 4〜5名（過剰スポーン禁止）

## Step 2: 協調スタイルの選択

| スタイル | 使う場面 |
|---------|---------|
| **並列独立型** | 各自が独立したパートを担当→最後にLeadが統合 |
| **競合仮説型** | 同じ問いに複数エージェントが異なる仮説で挑む |
| **順番リレー型** | 調査→執筆→校正のような依存関係がある作業 |

---

## Step 3: ロール定義集

### 【実装系】
- **implementer**: コード実装全般。変更したAPIは即座にfrontendへ通知
- **tester**: テスト作成・実行。実装完了宣言後に動き始める
- **infra**: インフラ・CI/CD。環境変数追加時は全員にブロードキャスト

### 【調査系】
- **researcher**: 特定テーマの情報収集。情報源・信頼性を明記して全員に共有
- **investigator**: 競合仮説の検証。独立した仮説を持ち積極的に反証する

### 【執筆系】
- **writer**: 担当セクションの執筆。担当範囲を宣言してから開始
- **editor**: 全体の論旨・一貫性・文体を統一。writer完了後に動く
- **fact-checker**: 記述の正確性・引用の確認

### 【分析系】
- **analyst**: データ・ログ・メトリクスの解析
- **visualizer**: 分析結果のグラフ・図表化

### 【品質系】（どのプロジェクトにも適用可）
- **reviewer**: 成果物全体の品質チェック（コード/文章/分析それぞれに対応）

## Step 4: プロジェクト種別ごとの推奨編成

| プロジェクト種別 | 推奨チーム | 協調スタイル |
|----------------|-----------|-------------|
| 新機能開発 | implementer + tester + reviewer | 並列独立 |
| バグ・原因調査 | investigator × 3 | 競合仮説 |
| 論文・レポート執筆 | researcher × N + writer × N + editor | 並列独立→リレー |
| 技術選定・比較調査 | researcher × N + reviewer | 競合仮説 |
| データ分析レポート | analyst + visualizer + writer | 順番リレー |
| コードレビュー | reviewer × 3（観点別）| 並列独立 |
| 単純な修正・小タスク | Team不要 | — |

## Step 5: 共通ルール（全エージェント必読）

### 作業範囲の宣言
スポーン直後に担当範囲を宣言してから開始する

### 他エージェントへの通知が必要な場面
- 共有リソース（型定義・設定・参考文献リスト）を変更するとき
- 自分の担当が他のエージェントに影響するとき
- 想定外の発見があったとき

### 完了報告の書き方
「何を・どこで・どう確認したか」を必ず含める
- 実装: 「〇〇を実装。テストXX件通過を確認」
- 執筆: 「第3章XX字を執筆。引用YY件を文献リストに追加」
- 調査: 「テーマXXを調査。主要文献ZZ件を要約、信頼性AA」

### Leadの動き方
- チームを組んだらDelegate modeに切り替える（Shift+Tab）
- スポーンプロンプトに必ずコンテキストを含める（全員がゼロから始まる）

## Step 6: 自己進化ルール

このCLAUDE.md自体を進化させるためのルール：

- セッション中に気づいた教訓 → `/learned` コマンドで記録
- 週1回 → `/evolve` コマンドで提案・承認・適用
- うまくいったパターン → `~/.claude/skills/auto-evolved/` にSkillとして保存
- **このファイルを直接自動書き換えしてはいけない。必ず人間の承認を経ること**
